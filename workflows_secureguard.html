SecureGuard AI — four key developer workflows shown as step-by-step flows

1

Developer saves file in VS Code

LSP client detects onDidSave event, extracts filePath + language

event: { uri: 'file:///src/routes/users.py', languageId: 'python' }
2

LSP client calls POST /scan

HTTP to localhost:3000, body: { filePaths, scanMode: 'fast', source: 'ide' }

latency target: <100ms to respond, scan runs async
3

scanner.ts runs Semgrep fast rules

Subprocess: semgrep --config p/python --json filePath. Gitleaks on same file.

timeout: 8s · if timeout → skip, no diagnostic shown
4

normalizer.ts maps raw output → Finding[]

Dedup by (filePath + lineStart + ruleId). Map Semgrep rule_id to CWE. Drop LOW severity in fast mode.

5

Decision: findings found?

No findings → return 200 empty array → LSP clears existing diagnostics for file

6

lsp.ts formats as LSP Diagnostic[]

severity: 1=Error (CRITICAL/HIGH), 2=Warning (MEDIUM). message = CWE name + one-line fix hint from registry.

{ range, severity, message, code: 'CWE-89', source: 'secureguard' }
7

VS Code renders inline squiggle + Problems panel entry

Developer sees red underline on line 23. Hovering shows message. No LLM called. Total: ~1–2s from save.

1

Developer runs git commit

Husky pre-commit hook fires before commit is written. Hook script: npx secureguard scan --staged

2

CLI resolves staged file list

git diff --cached --name-only --diff-filter=ACM → filePaths[]. Filter to .ts .js .py extensions.

scanMode: 'fast' · source: 'commit'
3

scanner.ts runs full fast suite on staged files

Semgrep (p/python + p/javascript) + Gitleaks + npm/pip audit for dependency CVEs

parallel subprocess execution · all three run concurrently
4

Decision: CRITICAL severity found?

No CRITICAL → exit code 0 → commit proceeds. MEDIUM/LOW → print advisory warning, commit proceeds.

5

CRITICAL found → block commit

exit code 1. cli.ts prints colored output to terminal with finding + quick fix hint from registry (no LLM).

CRITICAL [CWE-89] SQL Injection · users.py:23 · Fix: use parameterized queries
6

Developer fixes issue and re-commits

Or uses --no-verify to bypass (logged). Bypass rate is a key metric for your demo.

1

PR opened or new commit pushed

GitHub Actions triggers on: [pull_request]. Event: { action: 'opened'|'synchronize', pull_request: { number, head } }

2

Actions runner calls POST /pr on SecureGuard service

Body: { prNumber, repo, changedFiles[], baseRef, headRef, githubToken }

ANTHROPIC_API_KEY and GITHUB_TOKEN from repo secrets
3

scanner.ts runs deep scan on all changed files

Semgrep full ruleset (3000+ rules, p/owasp-top-ten + custom). Bandit for Python. Full dep audit. Cross-file dataflow.

scanMode: 'full' · timeout: 90s
4

remediator.ts calls Claude API for each HIGH/CRITICAL finding

buildPrompt(finding, registryEntry, codeContext) → claude-sonnet-4-6. Returns explanation + patch + testStub.

parallel calls · max 5 concurrent · stream to buffer
5

pr.ts posts inline review comments via GitHub API

POST /repos/{owner}/{repo}/pulls/{pr}/reviews with comments[]. Each comment: finding + AI explanation + code suggestion block (one-click apply).

6

policy.ts evaluates: BLOCK or PASS?

Any unresolved HIGH or CRITICAL → POST /repos/.../statuses with state: 'failure'. Checks branch protection — merge button grayed out.

status context: 'secureguard/policy-gate'
7

Developer applies fix → re-push → gate re-runs → PASS

Or security lead adds label 'security-override' → policy.ts sees label via GitHub API → posts passing status with override note in audit log.

1

Developer types @secureguard in Copilot Chat

GitHub Copilot routes message to registered extension endpoint. POST /copilot with Copilot skill payload.

{ messages: [...], context: { currentFile, selection, repoInfo } }
2

copilot route parses intent

Intent classifier (simple keyword match): 'scan file' → full scan + remediate. 'fix this' → remediate only on selection. 'explain' → registry lookup only.

3

scanner.ts scans current file or repo

For 'scan repo': spawn background job, return interim "scanning..." SSE event. For 'scan file': synchronous, return findings in <5s.

4

remediator.ts calls Claude with full context

Prompt includes: vulnerable snippet, surrounding function, file imports, registry guidance, company policy. Model: claude-sonnet-4-6, max_tokens: 1000, stream: true.

5

copilot.ts streams SSE response back to VS Code

Markdown formatted: severity badge → explanation → code block (fixed code) → OWASP reference → "Apply this fix?" button via sendPrompt.

Content-Type: text/event-stream · data: {token} chunks
6

Developer sees streamed response in Copilot Chat panel

Reads explanation, copies fix, or clicks "Apply this fix?" which triggers another @secureguard message to apply the patch via VS Code workspace edit API.
